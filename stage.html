Gitflow

GitFlow is a branching model for Git, created by Vincent Driessen. It has attracted a lot of attention because it is very well suited to collaboration and scaling the development team.

  • Parallel Development
  • Collaboration
  • Support For Emergency Fixes

What is Gitflow ?

The branches
  • Master
  • Develop
  • Feature / *
  • Release / *
  • Hotfix- *

What is a feature ?

New development (such as features and non-emergency bug fixes) is done in feature branches, and is only merged back into main body of code (develop) when the developer(s) is happy that the code is ready for go out onto a dev server or for review.

What is a release

A Release is a branch that is as the name suggest ready to deploy (in our proposal also pushed to UAT) this branch is merged back into develop and master on completion. Using a dedicated branch to prepare releases makes it possible for refinements on the current release that are not lost and don't stop working on features for the next release.

Whats a Hotfix ?

"hotfix" branches are used to quickly patch production releases. This is the only branch that should fork directly off of master. As soon as the fix is complete, it should be merged into both master and develop

Bamboo

Continuous Integration is the practice of testing each change done to your codebase automatically and as early as possible. But this paves the way for the more important process: Continuous Deployment.

Continuous Deployment follows your tests to push your changes to either a staging or production system. This makes sure a version of your code is always accessible.
