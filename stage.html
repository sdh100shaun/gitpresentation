What is Gitflow

Gitflow

Simple right not really a mind explosion.

GitFlow is a branching model for Git, created by Vincent Driessen. It has attracted a lot of attention because it is very well suited to collaboration and scaling the development team.

  • Parallel Development
  • Collaboration
  • Support For Emergency Fixes

The git flow branches

  1. Master
  2. Develop
  3. Feature / *
  4. Release / *
  5. Hotfix / *

What is a feature ?

New development (such as features and non-emergency bug fixes) is done in feature branches, and is only merged back into main body of code when the developer(s) is happy that the code is ready for release.
